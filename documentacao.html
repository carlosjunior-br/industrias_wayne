Introdução

Bem-vindo à documentação do sistema de gerenciamento de segurança das Indústrias Wayne. Este sistema foi desenvolvido para facilitar a gestão de recursos e usuários, garantindo a segurança e eficiência nas operações da empresa.

Requisitos do Sistema

  • Python 3.7 ou superior
  • FastAPI
  • Banco de dados (SQLite)
  • Node.js e npm (para o frontend)

Instalação

Para iniciar o sistema em modo local, siga as etapas abaixo:

  1. Navegue até a pasta do projeto.
  2. Crie um ambiente virtual em Python e instale as dependências necessárias executando o arquivo: setup.bat.
  3. Inicie o aplicativo com o comando: python -m app.main ou executando o arquivo iniciar aplicação.bat.

No primeiro acesso, você deverá criar um usuário Administrador. Siga as instruções apresentadas na interface para concluir a criação.

A estrutura do projeto é composta pelas seguintes pastas:

  • app: Código do backend.
  • front-end: Código do frontend.

Alimentação de Dados Iniciais

Para facilitar os testes e a utilização do sistema, um script Python foi desenvolvido para efetuar a alimentação automática da base de dados com dados pré-definidos. Isso permite que a aplicação seja testada com um conjunto de dados já existente, sem a necessidade de criar cada usuário e recurso manualmente.

A alimentação de dados iniciais é opcional e não uma exigência para a instalação ou funcionamento do sistema.

Para executar o script de alimentação de dados, siga as etapas abaixo:

  1. Certifique-se de que o servidor do banco de dados (backend) e o frontend não estejam em execução.
  2. Navegue até a raiz do projeto onde o arquivo cadastros_prontos.py está localizado.
  3. Execute o script com o seguinte comando no terminal:
    4. python cadastros_prontos.py
  4. Aguarde a execução do script. O terminal exibirá mensagens indicando o sucesso ou falha de cada cadastro.

Funcionalidades e Acesso ao Sistema

O sistema oferece uma variedade de funcionalidades que são acessadas de acordo com o papel do usuário, garantindo que cada um tenha acesso às informações e ferramentas necessárias para suas responsabilidades. As funcionalidades incluem:

  • Gestão de Recursos:

    Interface para gerenciar recursos internos, como inventário de equipamentos, veículos e dispositivos de segurança. Este módulo é acessível apenas para usuários com papéis de Gerente e Administrador.

  • Gestão de Usuários:

    Interface para gerenciar os usuários do sistema, permitindo a criação e modificação de contas. Este recurso está disponível exclusivamente para Administradores.

  • Dashboard de Visualização:

    Painel de controle que exibe dados relevantes sobre segurança, recursos e registros de acesso. Os dashboards possuem gráficos para demonstrar melhor os dados, proporcionando uma visualização mais clara e intuitiva. Este painel é acessível a todos os papéis de usuários.

Níveis de Acesso

Os níveis de acesso são definidos da seguinte forma:

  • Funcionário:

    Os funcionários têm acesso ao Dashboard, onde podem visualizar as informações relacionadas à empresa. O Dashboard é intuitivo e fornece uma visão geral dos recursos, usuários e acessos.

  • Gerente:

    Os gerentes possuem acesso ao Dashboard e à Gestão de Recursos, permitindo que administrem o inventário e equipamentos, garantindo que todos os recursos estejam disponíveis para as operações da empresa.

  • Administrador:

    Os administradores têm acesso total ao sistema, incluindo o Dashboard, Gestão de Recursos e Gestão de Usuários. Isso permite que gerenciem todos os aspectos do sistema, desde a criação de usuários até a supervisão completa dos recursos e segurança, monitorando o acesso e realizando ajustes conforme necessário.

Essas distinções de acesso são gerenciadas dinamicamente pelo sistema, garantindo que a interface se adapte automaticamente ao papel do usuário logado, promovendo uma experiência personalizada e segura.

Registro de Acesso

Todos os logins são registrados no sistema, permitindo que seja acompanhado quem acessou a plataforma e quando. Esse recurso é essencial para a segurança da informação e pode ser visualizado diretamente no Dashboard. Somente os administradores tem acesso à exclusão dos registros.

O gráfico de registros de acesso é exibido em linhas se forem até 31 dias de dados. Caso haja mais de 31 dias de dados, as informações são apresentadas em formato de barras por mês/ano, facilitando a visualização e análise dos registros.
